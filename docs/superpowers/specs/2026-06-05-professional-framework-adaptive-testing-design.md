# 专业心理学框架 + 自适应出题机制 设计文档

> 日期：2026-06-05
> 状态：待审批

## 1. 目标

将"精神状态鉴定中心"从趣味测试升级为具有心理测量学基础的人格评估工具，同时引入自适应出题机制，提升区分度和用户体验。

**核心要求**：
- 底层框架基于经过验证的心理学理论（大五人格）
- 题目保持鲁迅黑色幽默情境风格
- 自适应出题：不同用户拿到不同题目和题量
- 评分方法学严谨（Likert 量表、反向计分、效度检查）

## 2. 心理学框架

### 2.1 大五因子与现有维度的映射

保留现有 4 个维度作为结果解读层，底层用大五因子的 facet（子维度）构建题目。

| 现有维度 | 大五因子支撑 | 方向映射 |
|---------|------------|---------|
| 压力应对（抗争 vs 逃避） | 神经质（低→抗争）+ 尽责性（高→抗争） | 低焦虑、高自律 = 抗争 |
| 情绪管理（压抑 vs 宣泄） | 神经质（情绪稳定性）+ 开放性（感受性） | 低情绪性、高感受性 = 宣泄 |
| 社交模式（疏离 vs 连接） | 外向性（合群性）+ 宜人性（信任） | 高热情、高信任 = 连接 |
| 决策方式（理性 vs 感性） | 开放性（经验开放）+ 宜人性（共情） | 高幻想、高利他 = 感性 |

### 2.2 子维度拆分

#### 维度一：压力应对

| 子维度 | 大五来源 | 含义 | 题数 |
|-------|---------|------|-----|
| 焦虑倾向 | 神经质 → 焦虑 | 面对压力时的焦虑程度 | 3 题（1 题反向） |
| 压力韧性 | 神经质 → 压力敏感 | 能承受多大的压力 | 3 题 |
| 行动力 | 尽责性 → 自律 | 压力下是否主动应对 | 2 题 |
| 回避倾向 | 神经质 → 冲动性（反向） | 是否倾向于逃避 | 2 题（1 题反向） |

#### 维度二：情绪管理

| 子维度 | 大五来源 | 含义 | 题数 |
|-------|---------|------|-----|
| 情绪觉察 | 开放性 → 感受性 | 能否意识到自己的情绪 | 3 题 |
| 情绪抑制 | 神经质 → 冲动性（反向） | 是否压抑不表达 | 3 题（1 题反向） |
| 情绪释放 | 开放性 → 幻想 | 是否通过外在方式宣泄 | 2 题 |
| 情绪稳定性 | 神经质 → 情绪性 | 情绪波动幅度 | 2 题（1 题反向） |

#### 维度三：社交模式

| 子维度 | 大五来源 | 含义 | 题数 |
|-------|---------|------|-----|
| 社交能量 | 外向性 → 热情 | 社交后感到充电还是耗电 | 3 题（1 题反向） |
| 信任倾向 | 宜人性 → 信任 | 是否愿意向他人敞开心扉 | 3 题 |
| 孤独耐受 | 外向性 → 合群性（反向） | 独处时的舒适度 | 2 题 |
| 关系维护 | 宜人性 → 顺从 | 是否主动维系关系 | 2 题（1 题反向） |

#### 维度四：决策方式

| 子维度 | 大五来源 | 含义 | 题数 |
|-------|---------|------|-----|
| 逻辑依赖 | 开放性 → 概念化（反向） | 决策时是否依赖逻辑分析 | 3 题（1 题反向） |
| 共情驱动 | 宜人性 → 利他 | 是否因他人感受而改变决定 | 3 题 |
| 直觉倾向 | 开放性 → 幻想 | 是否相信"感觉对了" | 2 题 |
| 决策焦虑 | 神经质 → 犹豫（反向） | 是否反复纠结 | 2 题 |

## 3. 题目设计

### 3.1 格式规范

- **风格**：鲁迅黑色幽默情境题，有具体场景、有画面感、有荒诞感
- **选项数量**：4 / 5 / 6 个选项，按题目需要灵活选择
- **语言**：简体白话，保持现有叙事风格

### 3.2 选项与评分映射

| 选项数 | Likert 映射 | 说明 |
|-------|------------|------|
| 4 选项 | 1-4，按比例换算到 1-5 | 均值 = (raw - 1) × (4/3) + 1 |
| 5 选项 | 直接对应 1-5 | 无需换算 |
| 6 选项 | 两端极端 = 1 和 5，中间 4 项均匀分布 | 极端项算 1/5，中间按 2, 2.67, 3.33, 4 分布 |

### 3.3 反向计分

每个维度内设置 2-3 道反向计分题，选项顺序反转，防止模式化作答。反向计分公式：`score = (max + 1) - original_score`

### 3.4 效度检查题

3 道固定题目，随机插入测试中段。用于检测随意作答。例如："请选择'有点符合'"——如果用户选了其他选项，标记为无效作答。

### 3.5 题目示例

#### 压力应对 — 探测题

> 「项目被临时加了三个需求，deadline 不变。你坐在工位上，周围同事都在等你表态。你——」
>
> 1. 打开文档，开始列计划，假装一切尽在掌握
> 2. 先去倒杯水，回来再说
> 3. 打开招聘软件看了一眼，又关上了
> 4. 对同事说"没问题"，然后打开空白文档对着发呆
> 5. （6选项版本）……或者默默打开朋友圈，发了一条"奋斗"的鸡汤

#### 压力应对 — 分叉题（偏抗争分支）

> 「你确认领导的决策是错的，但其他同事都沉默了。你——」
>
> 1. 整理数据和案例，约领导单独谈
> 2. 在会上直接指出来，不管领导怎么想
> 3. 先执行，等结果证明你对了再说
> 4. 越级汇报
> 5. 发朋友圈阴阳"某些领导"

#### 压力应对 — 分叉题（偏逃避分支）

> 「deadline 快到了，但你还没开始。你知道自己在逃避。你——」
>
> 1. 设了十个闹钟，决定今晚通宵
> 2. 开始做最简单的部分，假装在推进
> 3. 跟领导说做不完，申请延期
> 4. 生病了（可能是真的）
> 5. 继续逃避，但焦虑已经开始影响睡眠了

#### 情绪管理 — 示例

> 「朋友当众开了一个让你很难堪的玩笑。饭桌上所有人都在笑。你——」
>
> 1. 完全不当回事，甚至还接了个梗
> 2. 假装不在意，回去之后越想越气
> 3. 当场怼回去，管他是不是朋友
> 4. 假装没听见，但整晚都没再说话
> 5. 散场后发了条朋友圈，仅他可见

#### 社交模式 — 示例

> 「周末，同事群有人提议聚餐。你看了看消息，然后——」
>
> 1. 立刻回复"去！"，已经开始想穿什么
> 2. 犹豫了五分钟，最后说"看情况"
> 3. 把消息标记已读，假装没看见
> 4. 开启免打扰，继续躺在沙发上
> 5. （6选项版本）……或者回复"去"，然后开始焦虑穿什么

#### 决策方式 — 示例

> 「两个工作机会摆在面前：A 稳定但无聊，B 有风险但有趣。你做了个 Excel 对比表，然后——」
>
> 1. 删掉 Excel，选了 B，因为"人生苦短"
> 2. 发了条朋友圈投票，结果出乎意料
> 3. 纠结了三天，最后问了五个朋友的意见
> 4. 选了 A，虽然你知道 B 更有趣
> 5. 把对比表发给爸妈，让他们帮你选
> 6. 到现在还没选，两个都快过期了

## 4. 自适应出题机制

### 4.1 架构选择

**题库 + 实时筛选**（方案 C）：所有题目打标签（维度、子维度、倾向、难度），运行时根据当前置信度从题库中实时挑选最合适的下一题。

### 4.2 题库元数据

每道题携带以下标签：

```json
{
  "id": "stress_07",
  "dimension": "stress",
  "subfacet": "anxiety",
  "polarity": "direct",
  "leaning": "fight",
  "difficulty": "boundary",
  "options": 5,
  "text": "题目文本",
  "scoring": { "stress": 3 }
}
```

字段说明：
- `dimension`：所属维度（stress / emotion / social / decision）
- `subfacet`：子维度（anxiety / resilience / action / avoidance 等）
- `polarity`：direct（正向计分）或 reverse（反向计分）
- `leaning`：倾向标签（fight / escape / suppress / vent / isolate / connect / rational / emotional / neutral）
- `difficulty`：easy（探测用）/ boundary（区分用）/ extreme（验证边界用）
- `options`：选项数量（4 / 5 / 6）
- `scoring`：评分映射

### 4.3 实时筛选算法

每道题答完后，系统执行：

1. **更新维度得分**：累加该维度的原始分
2. **计算置信度**：该维度已答题目的得分标准差，越小越确定
3. **检查覆盖度**：各子维度是否都已有至少 1 题
4. **挑选下一题**：从题库中筛选，优先级：

| 优先级 | 条件 | 说明 |
|-------|------|------|
| 1 | 低覆盖子维度 | 还没测到的子维度优先 |
| 2 | 中等难度（boundary） | 最有区分度的题优先 |
| 3 | 与当前倾向不一致 | 防止偏差累积，交叉验证 |

### 4.4 阶段切换

| 阶段 | 触发条件 | 行为 |
|------|---------|------|
| 探测 | 初始状态 | 每维度抽 1-2 题（easy 难度），快速建立基线 |
| 区分 | 某维度置信度未达标 | 选 boundary 难度题，针对低置信维度深入 |
| 确认 | 所有维度置信度达标 或 达到最大题数 | 补充 extreme 难度题验证边界 |
| 结束 | 置信度全部达标 或 总题数 ≥ 上限 | 进入效度检查 → 出结果 |

### 4.5 终止条件

- **理想情况**：置信度全部达标，约 21-25 题
- **上限保护**：最多 30 题（含 3 道效度检查），防止无限循环
- **提前终止**：如果连续 3 题得分完全一致且覆盖度已够，可以提前结束

### 4.6 题量弹性

- 最少：探测 6 + 区分 6 + 确认 6 + 效度 3 = **21 题**
- 最多：探测 8 + 区分 10 + 确认 9 + 效度 3 = **30 题**
- 每个用户的实际题量取决于置信度达标速度

### 4.7 数据流

```
用户答题
  ↓
更新维度得分 + 置信度
  ↓
筛选下一题（从题库中按优先级选）
  ↓
判断阶段切换
  ↓
重复直到终止条件满足
  ↓
效度检查（3 题）
  ↓
计算维度指数 → 原型映射 → 展示结果
```

## 5. 评分系统

### 5.1 原始分计算

每个维度的原始分 = 该维度所有已答题目的 Likert 分之和（反向题先转换）。

### 5.2 维度指数

维度指数 = 原始分按该维度已答题数标准化到 0-100。

公式：`index = (raw_score - min_possible) / (max_possible - min_possible) × 100`

### 5.3 原型映射

保持现有的 4-bit 原型映射系统不变：
- 指数 ≥ 50 → 该维度记为 1
- 指数 < 50 → 该维度记为 0
- 4 个维度组成 4-bit key → 查表得原型
- 特殊原型触发条件保持不变

### 5.4 效度检查处理

如果效度检查题作答异常（如全选同一选项、未按指示选择），在结果页提示"本次作答可能存在随意填写，结果仅供参考"。

## 6. 与现有系统的兼容

### 6.1 保留项

- 20 个人格原型（name、subtitle、typeCode、desc、guidance、insight）
- 原型映射逻辑（4-bit key + special_1~4）
- 结果页展示（灵魂肖像、人生指引、哲思、维度分析、综合指数）

- URL 参数分享（?r=xxxx）
- Canvas 墨点落字雨背景

- questions 数组 → 重构为题库（带元数据标签）
- answer() 函数 → 重写为自适应筛选逻辑
- showResult() → 适配新的评分算法
- 答题 UI → 增加进度指示（"第 X 题 / 约 Y 题"）
- 选项 UI → 支持 4/5/6 选项动态渲染

### 6.3 新增项

- 题库数据结构（questions pool with metadata）
- 置信度计算模块
- 实时筛选引擎
- 阶段状态机
- 效度检查逻辑

## 7. 实施范围

本次设计不包含：
- 题库的具体题目编写（需要大量内容创作，另行规划）
- 大五因子的标准化常模数据（趣味测试不需要）
- 跨语言适配
- 数据存储/用户追踪
